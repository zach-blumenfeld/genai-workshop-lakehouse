title: "Vega 2.4: Failed body control module logs configuration failure B2477 and dark cluster"
area: bulletins
---

# Condition

Some Vega 2.4 vehicles may present with a completely dark instrument cluster and multiple inoperative body electrical functions on startup. Scanning the network reveals DTC B2477 (module configuration failure) stored in the body control module, frequently accompanied by lost-communication codes from the cluster. The condition is often persistent rather than intermittent once it appears. See [Vega Electrical Service Manual :: Lost Communication with BCM and Cluster](doc://man-vega-electrical#network-communication-diagnostic-codes/lost-communication-with-bcm-and-cluster) for the related communication codes.

# Cause

The failure originates in the body control module, part BCM-9920, where corrupted internal configuration memory causes the module to fail its self-validation and set B2477. When this occurs the BCM stops broadcasting on the CAN bus, which silences the instrument cluster and triggers the dark-display symptom. Review [Vega Electrical Service Manual :: Body Control Module](doc://man-vega-electrical#electrical-system-overview/body-control-module) for module architecture and [Vega Electrical Service Manual :: Electrical System Overview](doc://man-vega-electrical#electrical-system-overview) for network topology.

# Repair Procedure

First run the communication diagnostic PROC-COMMS-DIAG to rule out a wiring or bus fault as the source of B2477, using [Vega Electrical Service Manual :: CAN Bus Communication Diagnosis Procedure](doc://man-vega-electrical#network-and-harness-sub-system-service/can-bus-communication-diagnosis-procedure). If the bus is healthy and the configuration failure persists, replace the body control module BCM-9920 and then perform configuration procedure PROC-BCM-CONFIG to load the correct vehicle calibration, per [Vega Electrical Service Manual :: Body Control Module Configuration Procedure](doc://man-vega-electrical#bcm-sub-system-service/body-control-module-configuration-procedure). Verify the cluster wakes and all codes clear before release. The analogous configuration-failure diagnosis on the related platform is described in [Kestrel Body Service Manual :: Module Configuration Failure Diagnosis](doc://man-kestrel-body#diagnosis/module-configuration-failure-diagnosis).
